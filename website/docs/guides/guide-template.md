# Guide Template

This template ensures all new guides follow a consistent format across the documentation.

---

## Purpose
Explain the goal of this guide in 2–3 sentences.  
- Why does it exist?  
- Who is the audience?  
- What problem does it solve?

---

## Prerequisites
List any tools, knowledge, or setup required before following this guide.  
- Example: Docker installed, access to NovaEco, API token.

---

## Steps
1. Step one — clear, actionable instruction.  
2. Step two — include code snippets or commands if needed.  
3. Step three — explain expected results.  

---

## Best Practices
- Keep instructions concise.  
- Use relative links to other docs.  
- Provide examples for clarity.  

---

## Outcomes
- **Positive:** Benefits or improvements expected.  
- **Negative:** Risks, limitations, or trade‑offs.
