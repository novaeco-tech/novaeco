# Use Case Template

This template ensures all new use cases follow a consistent format across Levels 1–4.

---

## Metadata
- **ID:** UC-XXX  
- **Level:** 1 (Onboarding) | 2 (Easy) | 3 (Medium) | 4 (Complex)  
- **Title:** Short descriptive name  
- **Actors:** Who is involved (user, developer, organization, enabler, sector, worker, product)  
- **Preconditions:** What must be true before this use case starts  

---

## Description
Explain the scenario in 2–3 sentences.  
- What happens?  
- Why is it important?  
- Which enablers/sectors/workers/products are used?

---

## Steps
1. Actor initiates action (e.g., user registers, developer generates API key).  
2. System responds (e.g., Identity service validates credentials).  
3. Data flows between enablers/sectors.  
4. Outcome achieved (e.g., dashboard updated, compliance verified).

---

## Outcomes
- **Positive:** Benefits, efficiency gains, sustainability impact.  
- **Negative:** Trade‑offs, risks, limitations.
